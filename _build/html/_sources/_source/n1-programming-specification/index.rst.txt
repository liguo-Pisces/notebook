为什么要执行严格的代码编写规范？
====================================

* 提高代码的可读性

遵守相同编码规范的代码，阅读起来会有统一的风格。具有相同的风格是降低学习曲线的有效方式。可读性的提高使别人理解它所需要的时间最小化。

* 提高代码的可维护性

这个可维护性是基于在代码的可读性上，好的源码往往都看上去很漂亮，很有美感。这里说的漂亮和美感不是指代码的逻辑清晰有条理，而是指感官上的视觉感受让人感觉很舒服。富有美感的代码让人赏心悦目，也容易让人读懂。

* 能为团队建立脚手架

在一个团队中，不是每一个成员，都拥有相同的知识体系，从而每个成员都有不同的能力上限。

建立合理的规范，可以建立团队在技术上的沟通的模式，为成员提供了有高度的脚手架，避免成员个体在程序设计上特定弱点所带来的质量短板。

团队中个人的优势能力决定了团队的质量能力的上限，而编码规范的执行，则决定了团队代码质量的下限。

**总结**

软件开发是一个多人协作的工作，如果没有合理的编码规范，最终导致的是代码混乱。从而导致一些列的问题出现，所以编程规范应该是每一个合格的程序员都需要掌握的能力。
